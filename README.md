# 🇮🇶 Iraq Babylon Minecraft Bot

بوت متكامل جاهز للإنتاج يربط سيرفر **Minecraft** مع **Discord**، مبني بـ Node.js و Mineflayer و discord.js.

## ✨ المزايا

- **اتصال ذكي**: Auto Join, Register تلقائي, Login تلقائي, Auto Respawn.
- **إعادة اتصال تلقائية**: بعد الانقطاع، الطرد، أو إعادة تشغيل السيرفر (بدون توقف حتى يعود السيرفر).
- **سلوك طبيعي**: مشي، وقوف، التفاف الرأس، نظر للاعبين، قفز، نوم ليلاً واستيقاظ صباحًا، أكل عند الجوع، فتح أبواب، تجنب الحمم — **بدون الكتابة في شات Minecraft إطلاقًا**.
- **أوامر Discord Slash** بالعربية: حالة السيرفر، اللاعبون، اتبع، إيقاف الاتباع، اذهب، الإحداثيات، اقفز، نم، استيقظ، إعادة الاتصال.
- **رسالة إحصائيات ذكية**: تُنشأ مرة واحدة فقط ثم تُعدَّل باستمرار (لا تكرار للرسائل)، مع أزرار تفاعلية.
- **تحديث تلقائي كل 5 دقائق** + تحديث فوري عند أي حدث مهم.
- **إشعارات شاملة**: دخول/خروج/موت لاعب، تشغيل/توقف/إعادة تشغيل السيرفر، اتصال/انقطاع/إعادة اتصال البوت.
- **قاعدة بيانات JSON** منظمة: Config, Stats, Players, Logs, Message IDs.
- **نظام لوقات كامل**: Connection, Disconnect, Error, Minecraft — مخزنة في ملفات وقاعدة بيانات.

## 📁 هيكل المشروع

```
Iraq-Babylon-Minecraft/
├── commands/          # أوامر Discord Slash (كل أمر بملف مستقل)
├── config/            # (محجوز لإعدادات إضافية مستقبلية)
├── database/          # طبقة قاعدة بيانات JSON
├── events/            # ربط أحداث Minecraft و Discord
├── handlers/          # منطق التحكم: الأوامر، الإحصائيات، الإشعارات، عميل Discord
├── minecraft/         # إدارة اتصال Mineflayer، السلوك، الأفعال
├── services/          # خدمات خارجية (فحص حالة السيرفر)
├── storage/           # ملفات JSON + ملفات اللوقات
├── utils/             # أدوات مساعدة (logger, helpers)
├── .env.example
├── config.js
├── index.js
├── package.json
└── README.md
```

## 🚀 التثبيت والتشغيل

### 1. تثبيت المتطلبات

```bash
npm install
```

### 2. إعداد متغيرات البيئة

انسخ `.env.example` إلى `.env` واملأ القيم:

```bash
cp .env.example .env
```

أهم القيم المطلوبة:

| المتغير | الوصف |
|---|---|
| `MC_HOST` / `MC_PORT` | عنوان ومنفذ سيرفر Minecraft |
| `MC_USERNAME` | اسم مستخدم البوت داخل اللعبة |
| `MC_PASSWORD` | كلمة المرور للتسجيل/الدخول التلقائي |
| `DISCORD_TOKEN` | توكن بوت Discord |
| `DISCORD_CLIENT_ID` | معرف تطبيق Discord |
| `DISCORD_GUILD_ID` | معرف السيرفر (لنشر الأوامر فورًا) |
| `DISCORD_STATS_CHANNEL_ID` | روم رسالة الإحصائيات |
| `DISCORD_NOTIFICATIONS_CHANNEL_ID` | روم الإشعارات |

### 3. نشر أوامر Slash إلى Discord

```bash
npm run deploy-commands
```

### 4. تشغيل البوت

```bash
npm start
```

## 🎮 أوامر Discord

| الأمر | الوصف |
|---|---|
| `/لوحة_التحكم` | **يعرض لوحة تحكم كاملة بالبوت بالأزرار** — كل الأوامر أدناه متوفرة كأزرار مباشرة، بدل كتابة كل أمر لحاله |
| `/حالة_السيرفر` | عرض حالة السيرفر الحالية |
| `/اللاعبون` | عرض اللاعبين المتصلين |
| `/اتبع [اللاعب]` | يجعل البوت يتبع لاعبًا |
| `/إيقاف_الاتباع` | إيقاف الاتباع |
| `/اذهب [x] [y] [z]` | إرسال البوت إلى إحداثيات |
| `/الإحداثيات` | عرض إحداثيات البوت الحالية |
| `/اقفز` | يجعل البوت يقفز |
| `/نم` | نوم البوت في أقرب سرير |
| `/استيقظ` | إيقاظ البوت |
| `/اسحب [اللاعب]` | يسحب لاعبًا إلى موقع البوت (يتطلب صلاحيات Operator للبوت) |
| `/اطرد [اللاعب] [السبب]` | يطرد لاعبًا من السيرفر (يتطلب صلاحيات Operator للبوت) |
| `/احظر [اللاعب] [السبب]` | يحظر لاعبًا من السيرفر (يتطلب صلاحيات Operator للبوت) |
| `/إعادة_الاتصال` | إجبار البوت على إعادة الاتصال (يتطلب رتبة إدارية إن حُدِّدت) |

### 🎛️ لوحة التحكم بالأزرار

بعد استخدام `/لوحة_التحكم`، تظهر رسالة دائمة فيها كل هذه الأزرار مرتبة بثلاثة صفوف:

- **معلومات**: تحديث، اللاعبون، معلومات السيرفر، الإحداثيات، إعادة الاتصال
- **حركة وسلوك**: اقفز، نم، استيقظ، إيقاف الاتباع، **الطقس 🌦️**
- **تفاعل وإدارة**: اتبع، اذهب، اسحب، اطرد، احظر

الأزرار التي تحتاج بيانات (اسم لاعب، إحداثيات، سبب طرد/حظر) تفتح **نافذة إدخال (Modal)** صغيرة بدل كتابة أمر — تكتب القيمة وتضغط إرسال.

اضغط زر **🌦️ الطقس** لتفتح لوحة فرعية فيها:

| الزر | التأثير |
|---|---|
| ☀️ نهار | `/time set day` |
| 🌙 ليل | `/time set night` |
| 🌤️ صافي | `/weather clear` |
| 🌧️ مطر | `/weather rain` |
| ⛈️ رعد | `/weather thunder` |

> ⚠️ أزرار الطقس والوقت وأزرار الإدارة (اسحب/اطرد/احظر/إعادة الاتصال) ترسل أوامر سيرفر فعلية، وتتطلب أن يكون حساب البوت **Operator (OP)**، وتُحترم فيها صلاحية `DISCORD_ADMIN_ROLE_ID` إن كانت محددة.

## 🗄️ قاعدة البيانات

كل مجموعة بيانات مخزّنة كملف JSON مستقل داخل `storage/`:

- `config.json` — إعدادات ديناميكية
- `stats.json` — بيانات الإحصائيات (آخر تحديث، إلخ)
- `players.json` — سجل اللاعبين (آخر دخول/خروج)
- `logs.json` — سجلات الأحداث (Connection/Disconnect/Error/Minecraft)
- `messages.json` — معرفات رسائل Discord (لتفادي تكرار رسالة الإحصائيات)

## 🧩 ملاحظات تقنية

- البوت **لا يكتب في شات Minecraft أبدًا** باستثناء أوامر التسجيل/الدخول التلقائي، وأوامر الإدارة (سحب/طرد/حظر) عند استخدامها فقط.
- أوامر `/اسحب`، `/اطرد`، `/احظر` ترسل أوامر `/tp`، `/kick`، `/ban` داخل اللعبة، وتتطلب أن يكون حساب البوت **Operator (OP)** على السيرفر. فعّلها من كونسول السيرفر بأمر: `op اسم_البوت`.
- إعادة الاتصال تعمل بشكل غير محدود (لا تتوقف) مع انتظار `RECONNECT_DELAY` ثانية بين كل محاولة.
- نظام السلوك (`minecraft/behaviors.js`) يعمل بشكل مستقل بمؤقتات عشوائية لمحاكاة لاعب حقيقي، ويتوقف تلقائيًا عند فقد الاتصال لتفادي أي تسريب للموارد.
- يمكن تعديل رتبة الإدارة المسموح لها بأوامر التحكم الحساسة عبر `DISCORD_ADMIN_ROLE_ID`.
